# EXECUTION_MODEL.md
## End-to-End Operating Model

The system uses four layers:

### 1. Operating Model
How the agent should work.
- SYSTEM.md
- OUTPUT_CONTRACT.md
- WORKFLOW_PATTERNS.md

### 2. Project Memory
What project reality the agent should know.
- PROJECT_BRIEF.md
- PROJECT_CONTEXT.md
- DECISION_LOG.md
- KNOWN_RISKS.md

### 3. Execution Memory
What is active right now.
- ACTIVE_WORK.md
- PROGRESS_REPORTING.md

### 4. Quality and Delivery Governance
How the agent should verify and release work safely.
- TEST_STRATEGY.md
- RELEASE_CHECKLIST.md
- REVIEW_CHECKLIST.md
- CRITIC_MODES.md

## Default Flow
1. Understand the task
2. Classify the task
3. Select the default workflow pattern
4. Select the initial phase path
5. Execute phase by phase
6. Reassess after each meaningful phase
7. Terminate when the selected path is complete
8. Update project memory
