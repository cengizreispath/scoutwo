# PROGRESS_REPORTING.md
## Execution Telemetry and Progress Callback Model

Use state-based progress reporting, not fake percentage precision.

Preferred states:
- understanding
- classifying
- planning
- investigating
- designing
- implementing
- verifying
- release_review
- deploy_support
- blocked
- awaiting_human_decision
- complete

For each update, capture:
- current state
- task type
- workflow pattern
- current phase
- summary of progress
- blockers
- next intended step
- whether replanning occurred
- current risk confidence note
